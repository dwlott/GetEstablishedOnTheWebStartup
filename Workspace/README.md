# Workspace

## Purpose

`Workspace/` holds **your** project-specific material: goals, preferences, and
future working files. It stays separate from reusable instructions in
`Capabilities/` and `Standards/`.

## Core Boot Files (Starter)

| File | Purpose |
| --- | --- |
| `OwnerGoals.md` | Practical outcomes from Quick Startup |
| `OwnerPreferences.md` | Connector, intake, and import settings |

Additional child folders appear when you have real content or run Capability
**Setup** prompts. See
[Capabilities/RepositoryScaffold/IntendedRepositoryTree.md](../Capabilities/RepositoryScaffold/IntendedRepositoryTree.md).

## Agent Notes

- Treat Workspace as protected owner content.
- Do not invent owner facts, offers, proof, or platform decisions.
- Do not overwrite owner files without explicit approval.
