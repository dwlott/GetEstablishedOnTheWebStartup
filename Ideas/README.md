# Ideas

Formal repository term chain: **Goals** → **Ideas** → **Plans** →
**Capabilities** (→ **Skills** when executable).

| Term | Meaning |
| --- | --- |
| **Goals** | What matters—**Repository Goals** (core set in `Plans/GoalsIdeasPlansCapabilitiesModel.md`) and **Owner Goals** (Quick Startup, refined as the owner works) |
| **Idea** | One tracked possibility (row in `Plans/Ideas.md`) |
| **Ideas** | This folder and the register in `Plans/Ideas.md` |
| **Plans** | Structured work from selected Ideas toward Goals |

Captured possibilities live here before they become approved Plans, reusable
Capabilities, or Standards.

| Layer | Location | Role |
| --- | --- | --- |
| Owner Goals | `Capabilities/GettingStarted/` (Quick Startup) | Owner's practical outcomes; `Workspace/OwnerGoals.md` when durable |
| Raw capture | `Ideas/FutureIdeas.md` | Low-friction parking lot |
| Tracked Idea | `Plans/Ideas.md` | One row per mature Idea |
| Plan | `Plans/` | Selected Ideas shaped into action |

See [GoalsIdeasPlansCapabilitiesModel.md](../Plans/GoalsIdeasPlansCapabilitiesModel.md)
and [RepositoryGoals.md](../Plans/GoalsIdeasPlansCapabilitiesModel.md).
