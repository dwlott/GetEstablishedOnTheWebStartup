# Repository Goals

This file is the North Star for **Repository Goals** — the core, durable goals
for how this learning repository stays organized, indexed, reusable, and safe.

Repository Goals are **separate** from **Owner Goals**
([../Workspace/OwnerGoals.md](../Workspace/OwnerGoals.md)), which capture the
practical outcomes a specific owner wants. Owner Goals change per project;
Repository Goals are stable across owners who adopt this starter.

## How To Use

- Read this file when you need the durable source of truth for how the
  repository itself should behave.
- Do not overwrite these with project-specific outcomes — those belong in
  `Workspace/OwnerGoals.md`.
- Refine only when the repository's operating model genuinely changes.

## Core Repository Goals

| Goal | Why it matters |
| --- | --- |
| Keep one clear source of truth | The local Git repository is authoritative; avoid conflicting copies. |
| Give humans and agents a reliable read-first path | `AGENTS.md` → `README.md` → this file and the search map. |
| Capture Ideas before they disappear | Route raw ideas into `Ideas/` and the curated register in `Plans/Ideas.md`. |
| Shape selected Ideas into Plans before rushing into tools | Follow [GoalsIdeasPlansCapabilitiesModel.md](GoalsIdeasPlansCapabilitiesModel.md). |
| Promote only repeatable work into Capabilities | Keep `Capabilities/` reusable, not one-off. |
| Keep files modular and indexed | Simple metadata so future agents find the right context. |
| Preserve human control | Reviewable changes, batched questions, clear handoffs. |
| Expand only when there is a real need | Avoid speculative structure. |
