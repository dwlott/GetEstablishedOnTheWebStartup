# Repository Instructions

This is **GetEstablishedOnTheWebStartup** ŌĆö a web-presence starter repository
for goals, ideas, plans, website content drafts, and local profile planning.

It is **not** the GetEstablishedOnTheWeb product repository and **not** the
GetEstablished method host. After you adopt this folder, **this workspace** is
the source of truth for your work.

## Quick Start

| Situation | What to do |
| --- | --- |
| First time in this repo | Say **"Begin Quick Startup from AGENTS.md"** ŌĆö follow `Capabilities/GettingStarted/GettingStarted.md` |
| Review website page examples | **ContentReview** on `Content/Website/Pages/` |
| Draft your one-page business site | **OnePageWebsite** |
| Google or Yelp profile planning | **GoogleBusinessProfile** / **YelpBusinessProfile** (Planned) |
| Git status, commit, push | **GitHubWorkflow** (owner must ask for commit/push) |
| Mirror refresh to Dropbox or GDrive | **MirrorToWindows** ŌĆö reads `Workspace/OwnerPreferences.md` |
| Not sure what's going on | **SituationalAwareness** Rules |

Before acting: read this file, then
[Standards/AgentSituationalAwareness.md](Standards/AgentSituationalAwareness.md).

## Primary Goal Of Repository

Help the owner **capture Ideas**, turn selected Ideas into **Plans**, and carry
Plans through to **desired positive outcomes** ŌĆö with agents assisting in small,
reviewable steps.

Formal chain: **Goals** ŌåÆ **Ideas** ŌåÆ **Plans** ŌåÆ implemented work.

**Web presence** is a supported outcome: example page drafts under
`Content/Website/Pages/`, one-page business website drafts, and future Google/Yelp
profile guidance Capabilities.

## Starting Rule

Before making changes, read this file first.

The active local repository root is **this folder** ŌĆö for example:

```text
C:\Repositories\GetEstablishedOnTheWebStartup
```

Or your chosen project path after download. Do not assume the root is
`C:\Repositories\GetEstablishedOnTheWeb`.

When the owner references **`agents.md`**, read **`AGENTS.md`** at the repository
root (case-insensitive intent).

## First 10 Minutes (Agent Startup)

On every new pass, before editing:

1. Read this file (`AGENTS.md`).
2. Read [Standards/AgentSituationalAwareness.md](Standards/AgentSituationalAwareness.md).
3. Read [Plans/RepositorySearchMap.md](Plans/RepositorySearchMap.md) for placement.
4. If resuming work, read [Plans/AgentTaskBacklog.md](Plans/AgentTaskBacklog.md) for **Ready Next**.
5. Route the task: read [Capabilities/RouterIndex.md](Capabilities/RouterIndex.md) first,
   confirm in [Capabilities/AgentCapabilityGuide.md](Capabilities/AgentCapabilityGuide.md),
   then the owning Capability's `Rules.md` and `Prompt.md`.
6. Confirm scope with the owner when unclear; do not expand scope silently.
7. Edit **local Git only** unless cloud review sync was explicitly requested.
