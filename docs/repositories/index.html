Repositories

Start with the repository package this site is built around: GetEstablishedOnTheWebStartup. It is the full web-presence workspace. If you do not want or need the web and WordPress Capabilities, GetEstablishedStartup is the lighter core choice.

Each package is a GetEstablished repository: modular, fully indexed, human-readable Markdown with Capabilities your AI agent can read and route to.

What every package gives you

Both packages share the same harmonized core, so you get the same dependable foundation either way:

  • A clean, fully indexed folder structure an agent can navigate without getting lost.
  • A simple path from idea to done: Goals → Ideas → Plans → implemented work.
  • Capabilities — packaged workflows, each with Rules, a Prompt, and an index so the next pass starts faster than the last.
  • Begin Quick Startup — your first session captures your Owner Goals and points you at the first tasks.

Get Established On The Web — web-presence platform

Get Established On The Web is the primary repository package: the same dependable core plus the web Capability group for listings, page content, and WordPress.

Best when your project is specifically about getting established on the web. Pairs with the practical checklists on Get Listed.

Download GetEstablishedOnTheWebStartup (ZIP) (~450 KB) — full details on the Get Established On The Web page.

GetEstablished — lighter core platform

GetEstablished is the lighter core repository: the universal GetEstablished method for any project, without the web and WordPress Capability group.

Choose it if you want to try the method first, or if your project is not about getting listed, building pages, or publishing with WordPress. GetEstablished is modular and fully indexed by design, so your agent finds the right context quickly and spends fewer tokens as you work.

Download GetEstablishedStartup (ZIP) (~2 MB) — full details on the GetEstablished page.

How To Choose

  • Web presence — listings, site content, WordPress → Get Established On The Web, the full web-presence package.
  • Try the method first or skip web workflows → GetEstablished, the lighter core platform.

The core is shared. GetEstablishedOnTheWebStartup stays in the forefront because it is the site’s main promise; GetEstablishedStartup is the lighter choice when you do not want the web Capability group.
