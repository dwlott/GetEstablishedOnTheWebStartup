Get Established On The Web

GetEstablishedOnTheWebStartup is the primary repository package for this site — the web-presence platform with the core GetEstablished method plus the web Capability group for listings, page content, and WordPress workflows.

Use it when your project is specifically about getting established on the web.

Download

Download GetEstablishedOnTheWebStartup (ZIP) (~450 KB)

  1. Download the ZIP and unzip it to a stable folder (not Downloads).
  2. Open the folder in your favorite AI platform.
  3. Run Begin Quick Startup from AGENTS.md.

Not sure yet, or do not need the WordPress Capabilities? GetEstablishedStartup is the lighter core option you can try first.

Capabilities

Capabilities are the interesting part — packaged workflows your agent reads and follows. GEOTW includes the full core platform plus the web Capability group.

Standouts in this package:

  • Situational Awareness — orient every agent pass before work begins.
  • Getting Started — Begin Quick Startup writes your Owner Goals and first tasks.
  • Web Presence Node — plan and track one listing or profile at a time across Google, Bing, Facebook, Nextdoor, Foursquare, BBB, Yelp, Apple Maps, and Yellow Pages.
  • One Page Website — scaffold and grow site content from Markdown.
  • Content Review — review drafts before they go public.
  • Chat Memory Capture — turn useful chat into durable repository files.

The Capability posts below list everything in GetEstablishedOnTheWebStartup. Browse the full Capability catalog for every public module on this site.

GetEstablished vs Get Established On The Web

Both repositories share the same harmonized core. GetEstablishedOnTheWebStartup stays in front because it adds the web modules listed above and room to grow into listings and WordPress workflows.

Need the universal method only? GetEstablished is the lighter core platform.
