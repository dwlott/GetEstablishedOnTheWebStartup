# Website

## Purpose

This folder holds source material for the future Get Established On The Web website.

## What Belongs Here

- Page drafts and website content source files.
- Future website content planning artifacts that are specific to site content.
- Framework-neutral materials that can feed a later website build.

## What Does Not Belong Here

- Website framework scaffolds before the build approach is chosen.
- Deployment setup.
- Generated site output.
- General project plans that belong under `Plans/`.

## Important Folders

- `Pages` - current plain Markdown page drafts.

Expected future files may include content blocks, navigation notes, or asset inventories when they become useful.

## Agent Notes

Do not build the website from this folder until the project explicitly enters a website build phase.
