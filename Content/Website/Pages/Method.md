# The Get Established Method

**Get Established On The Web** is the public home for practical web help, repository
offers, and the **GetEstablished** method — the bedrock workflow behind every package.

## What GetEstablished is

GetEstablished is a **learning repository pattern**: capture **Ideas**, shape **Plans**,
implement outcomes, and turn repeatable work into **Capabilities** agents can route to.

You do not need to read the method host repository to get started on Google or Yelp.
When you want the full story — how files, GitHub, and AI assistants work together —
you are in the right place.

## The formal chain

```text
Goals → Ideas → Plans → implemented work → Capabilities (when work repeats)
```

- **Repository Goals** keep the system organized and reusable.
- **Owner Goals** (your outcomes) live in your workspace, not in generic templates.
- **Capabilities** are durable workflow modules — each with a clear purpose and rules.

Browse the **[Capability catalog](Capabilities.md)** for visitor-friendly summaries of
each public module. Agent Rules stay in Git; the website explains *what* each Capability
helps with and *when* to use it.

See **[GetEstablished](GetEstablished.md)** for the **free** GetEstablishedStartup package
and which Capability posts it includes. See **[Offers](Offers.md)** for the paid
**GetEstablishedOnTheWebStartup** web repository and future specialty products.
