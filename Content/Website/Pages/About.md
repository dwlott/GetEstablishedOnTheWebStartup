# About

Get Established On The Web helps people, businesses, bands, and projects **get established on the web** — with practical basics first and a deeper repository method when you are ready.

## The Practical Mission

Most visitors need common-sense help: Google Business Profile, Yelp, accurate
business information, and a clear path before they commit to themes, hosting, or
complex tools.

This site leads with that work. The **GetEstablishedOnTheWeb** repository and
**GetEstablishedOnTheWebStartup** package are the unique offering — an AI-ready,
GitHub-ready way to keep planning, content, prompts, and Capabilities organized
as your web presence grows.

## The Repository Packages

**GetEstablishedOnTheWebStartup** is the main package for this site: an AI-ready,
GitHub-ready workspace with web-specific Capabilities for listings, content, and
WordPress workflows as those modules mature.

**GetEstablishedStartup** is the lighter core download for people who want to try
the method first, or who do not need the web and WordPress Capability group.

Both packages share the GetEstablished core method: capture Ideas, shape Plans,
implement work, and reuse proven workflows as **Capabilities**.

Capabilities stack on top so each offer stays focused. A book author does not
need Yelp workflows; a local business does.

## What Makes This Site Different

GetEstablishedOnTheWeb.com is both:

- A **practical guide** for web presence basics anyone can use.
- A **showcase for the GetEstablished repository model** — organized files, agent
  instructions, reviewable changes, and harmonizable Capabilities on GitHub.

AI assists the workflow. It does not replace your judgment or your decisions.

## Ecosystem (Harmonized, Separate Sites)

Some products have their own websites and companion repositories built on the
same GetEstablished core:

- **MoverCalcs** — web funnel case study; movercalcs.com is a separate site.
- **MetaToolExcel** — metatoolexcel.com is a separate site.
- **GithubAgenticAICertPrep** — planned companion repo when fully developed.

GEOTW explains how they fit the GetEstablished story. It does not host those apps.

## Long-Term Direction

The project aims to offer **downloadable Capabilities** on GitHub so owners can
share and harmonize workflow knowledge — plus guided setup and AI-assisted paths
that can sustain the work gracefully. No hype: progress is shown through real
repository milestones and open questions, not invented testimonials.

## How The Project Should Feel

Practical. Calm. Clear. Capable.
