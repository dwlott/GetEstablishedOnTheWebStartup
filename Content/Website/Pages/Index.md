# Website Page Drafts

These files are source drafts for the future Get Established On The Web website.

They are not a website build, theme, framework, or deployment setup. Keep them in plain Markdown, which is simple text with lightweight formatting, until the project has chosen a minimal publishing approach.

The website and repository should work together. The website explains the method, and the repository holds the source-of-truth files: plans, prompts, page drafts, open questions, reviews, and next tasks.

## Draft Pages

- `Home.md` - first-page message and project promise.
- `GetListed.md` - inspirational hub linking to practical listing guides.
- `UseAI.md` - using GetEstablished with AI agents (Quick Startup entry).
- `GoogleBusinessProfileBasics.md` - researched basics and official links for getting established with Google Business Profile.
- `YelpBasics.md` - researched basics and official links for getting established on Yelp.
- `AppleMapsBasics.md` - Apple Business / Apple Maps listing basics.
- `YellowPagesBasics.md` - YP.com listing basics.
- `StartHere.md` - mothballed; harmonized into Get Listed and Use AI.
- `HowItWorks.md` - mothballed method workflow page.
- `Offers.md` - mothballed repository offers page.
- `Method.md` - mothballed GetEstablished method page.
- `About.md` - why the project exists and how the repository proves the method.

## Intended Visitor Flow

- New visitors should understand the promise on `Home.md`.
- Beginners should move to `GetListed.md` and pick a listing platform.
- Users ready for AI-assisted work should read `UseAI.md`.
- Local-business readers use the listing basics pages as checklists with official links.
- Readers who want the reason behind the project should read `About.md`.

## Link Intent

When these drafts become website pages, the first navigation path should be:

- `Home.md` to `GetListed.md`.
- `GetListed.md` to listing basics pages (Google, Yelp, Apple Maps, Yellow Pages).
- `GetListed.md` or listing basics pages to `UseAI.md` when ready for repository + AI depth.
- `UseAI.md` to `AgentPermissions.md` and `GitHubWorkflow.md` when needed.
- `About.md` from the main navigation or footer.

Source-resource links in the starter workspace point to:

- `../../../Plans/UserSetupGuide.md` for the beginner setup path.
- `../../../Capabilities/LocalAgentToolSetup/AgentSandboxAndPermissions.md` for detailed sandbox and permissions guidance.
- `../../../Standards/TemplateGuidance.md` for copying the method.
- `../../../Capabilities/GettingStarted/UserDiscoveryPrompt.md` for the first discovery step.
- `../../../Plans/OpenQuestions.md` for non-blocking decisions.

Public site CTA: **Get Established** (starter workspace download).

## Drafting Rules

- Keep the tone practical, credible, calm, and business-focused.
- Keep claims grounded in what the repository can demonstrate.
- Preserve the idea that this repository is built with the same method it teaches.
- Explain repository-specific terms when a beginner may not know them yet.
- Do not add framework-specific syntax unless the website approach has been chosen.
