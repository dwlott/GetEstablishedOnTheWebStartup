# Offers

> **Starter example:** product offer copy from building **GetEstablishedOnTheWeb**.
> Adapt or replace for your business.

Get Established On The Web promotes practical web establishment first. The main
repository offer is **GetEstablishedOnTheWebStartup**. For people who do not want
the web or WordPress Capability group, **GetEstablishedStartup** is the lighter
core option.

## Repository Offerings

| Offer | Posture | Best for |
| --- | --- | --- |
| **GetEstablishedOnTheWebStartup** | **Primary web package** | Getting established **on the web** — Google, Yelp, site content, WordPress / Genesis / Altitude workflows |
| **GetEstablishedStartup** | **Lighter core option** | Try the method first, or use the core for a non-web project |
| **Specialty repositories** | **Paid** (as released) | MoverCalcs, MetaToolExcel, certification prep, and future product-specific repos |

Both startup packages share **harmonized core** GetEstablished instructions — like
shared Genesis patterns across StudioPress products. Including core in the paid web
package is **composition**, not rank.

Hub pages: `/get-established-on-the-web/` for the primary web package and
`/getestablished/` for the lighter core option.

## Primary Product — GetEstablishedOnTheWebStartup

**GetEstablishedOnTheWebStartup** is the main repository package on this site:

- Harmonized GetEstablished core **plus** web-specific Capabilities.
- Pairs with the practical Google and Yelp basics on this site.
- Includes WordPress, Genesis, and Altitude operating knowledge (local product
  repo ships this workflow today).
- AI-ready folder structure for plans, prompts, content drafts, and reviewable agent tasks.
- GitHub-ready when you connect your repository.

Purchase path, pricing, and packaging are not published yet — no invented prices or dates.

### What the web repository helps you create

- Accurate Google and Yelp presence checklists tied to your workspace.
- Identity, audience, and offer notes in plain files.
- Framework-neutral website page drafts before platform lock-in.
- A backlog of next tasks agents and you can work through safely.
- A path from basics to WordPress publishing with Genesis and Altitude.

## Lighter Core — GetEstablishedStartup

**GetEstablishedStartup** is the try-it-first repository:

- Goals → Ideas → Plans → implemented work → Capabilities.
- Quick Startup and universal Capabilities for any strong repository.
- No web or WordPress Capability group.
- Enough for many projects that only need the organized core.

Download URL (GitHub template or ZIP) will appear here when the owner publishes it.
No purchase required.

## Downloadable Capabilities on GitHub

Long-term, GetEstablished aims to offer **harmonizable Capabilities** — reusable
workflow modules owners can download, adopt, and improve on GitHub. GEOTW is the
public face of that story for web presence work. Community model and release
timing remain open.
