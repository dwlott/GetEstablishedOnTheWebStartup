# How It Works

Plan, organize, review, and grow — with practical web basics first.

Getting established on the web is a sequence, not a single launch day. Here is the path from first plan to published site, and where the practical basics, the workspace, and AI assistance each fit. Choose your path first; the steps after it apply once you are ready for the full workflow.

## Step 0 — Choose Your Path

**Path A — Basics only:** Use the Google and Yelp checklists on this site. No
repository required.

**Path B — Full web workflow:** Use **GetEstablishedOnTheWebStartup** (when
published) to keep plans, content, prompts, and Capabilities in one AI-ready
workspace built on **GetEstablished**.

**Path C — Core only:** **GetEstablishedStartup** for non-web projects (no web
Capability modules).

Most business and band visitors start with Path A, then move to Path B.

## 1. Plan

Clarify identity, audience, offer direction, and first pages. For local
businesses, include Google Business Profile and Yelp in the plan early — not
after the website launch.

## 2. Write

Draft source content in plain Markdown. Start with what a visitor must understand
first. Platform choice (WordPress, etc.) can wait until the message is clear.

## 3. Organize

Use predictable folders for plans, prompts, content, and open questions.
**GetEstablishedOnTheWebStartup** provides the structure; GitHub holds history
when you connect a remote.

## 4. Review

Improve in small passes. AI agents work from repository files and instructions —
you approve changes. Nothing ships silently.

## 5. Build

When content is ready, build the public site. GetEstablishedOnTheWeb.com uses
WordPress with Genesis and Altitude locally; your business site may follow the
same stack or another path after your content is ready.

## 6. Publish And Improve

Launch when approved. Keep the repository as source of truth; sync public copy
deliberately. Add Capabilities (Google, Yelp, WordPress modules) as they mature.

## Capability Routing

Each task maps to a focused **Capability** — for example, Google Business Profile
basics belong to a `GoogleBusinessProfile` Capability. As Capabilities grow, the
workspace keeps pointing each piece of work to the right one, so you are never
guessing what to do next.
