# One-Page Business Website Content

## Purpose

This folder holds plain Markdown source content for the default simplest website output: a one-page business website draft.

It supports the repository-backed workflow by keeping practical business website inputs visible before any website build, publishing platform, framework, dependency, or automation is chosen.

The Business Plan Helper can eventually use this content set as a practical output target: collect business basics, review open decisions, then turn approved inputs into a simple one-page website draft.

## What Belongs Here

- One-page business website input fields.
- Recommended one-page website sections.
- Draft source content that could later be copied, converted, or adapted into a simple public website.
- Notes that help agents understand what business information is needed before publishing.

## What Does Not Belong Here

- HTML files.
- Website framework code.
- WordPress configuration.
- Deployment setup.
- Generated website output.
- Final pricing, legal, financial, or proof claims that the user has not approved.

## Important Files Or Expected Files

- `OnePageWebsiteInputs.md` - practical fields to collect before drafting the one-page website.
- `OnePageWebsiteSections.md` - recommended section order and content purpose.

Future files may include a filled-in one-page website draft after real user input exists.

## How To Use These Files

1. Start with `OnePageWebsiteInputs.md`.
2. Fill in the essential fields first.
3. Leave optional or unknown fields blank.
4. Move non-blocking human decisions to `Plans/OpenQuestions.md`.
5. Use `OnePageWebsiteSections.md` to turn approved inputs into a one-page draft later.
6. Review the draft before any publishing or platform decision.

Do not invent testimonials, proof, pricing, legal claims, policy language, credentials, service areas, or contact details.

## Agent Notes

Keep this folder plain Markdown and framework-neutral.

Use it as source content only. Do not build, publish, choose a platform, add dependencies, or add automation from this folder unless explicitly instructed.

Child pages are a future expansion. Start with the one-page version first.

A later pass may add `OnePageWebsiteDraft.md` after real input exists. Do not create that draft just to fill space.
